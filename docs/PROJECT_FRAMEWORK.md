# 家庭工具箱移动端 Web 项目框架（苹果风）

## 项目概览
- 目标：构建一个简约、现代、苹果风的移动端 Web 应用，用于家庭工具箱场景（记账、维修、保养、待办、成员等）。
- 技术栈：
  - 前端：Vue 3 + Vite + TypeScript、Vant（移动端 UI）、UnoCSS/Tailwind（实用类样式）、Pinia（状态）、Vue Router（路由）、Iconify+Tabler（图标）、@vueuse/motion（微动效）、vite-plugin-pwa（PWA）。
  - 数据库：MySQL 5.7。
- 风格：苹果风（留白充足、细线图标、系统蓝强调、半透明毛玻璃、柔和阴影、统一圆角、克制微动效）。

## 前端目录结构（建议）
```
frontend/
  ├─ index.html
  ├─ vite.config.ts
  ├─ package.json
  └─ src/
      ├─ app/
      │   ├─ main.ts            # 应用入口
      │   ├─ App.vue            # 根组件
      │   ├─ router/            # 路由定义（路由懒加载）
      │   └─ store/             # Pinia 状态（用户、设置、缓存）
      ├─ pages/                 # 页面（首页/工具/日程/我的）
      ├─ components/            # 通用组件（按钮/卡片/表单项/弹窗）
      ├─ widgets/               # 业务小组件（工具卡、记录条目等）
      ├─ services/
      │   ├─ api/               # 接口封装（Axios 或 OpenAPI 生成）
      │   └─ auth/              # 鉴权与刷新逻辑
      ├─ styles/                # 设计令牌（颜色/圆角/阴影）、全局样式
      ├─ assets/                # 静态资源（图标、图片）
      └─ pwa/                   # PWA 配置与离线兜底页
```

## 苹果风主题规范（简版）
- 字体栈：`-apple-system, BlinkMacSystemFont, Segoe UI, Roboto, Inter, Helvetica Neue, Arial, sans-serif`。
- 主色：系统蓝（近似 `#007AFF`），仅用于强调与交互；辅助色克制。
- 灰阶：中性灰（如 `#8E8E93` 系列），减少纯黑纯白高对比。
- 圆角：卡片/弹窗统一圆角（建议 12px）；输入控件更小（6–8px）。
- 阴影：轻柔、低透明度，用于层级区分，避免强烈浮动感。
- 毛玻璃：顶部/底部栏可使用适度 `blur + saturate`，保持内容可读性。
- 间距：统一 8px 间距体系（8/16/24/32）。

## 微动效基准
- 时长：150–250ms；页面层级过渡可 250–300ms。
- 曲线：`ease-out` 为主，少量轻弹性（不夸张）。
- 场景：
  - 按钮/图标：按压轻微缩放（0.96→1.0）、淡入淡出、阴影收敛。
  - 列表/卡片：入场交错、悬停轻提（y=-2，shadow+）。
  - 页面切换：方向语义一致（前进右进、后退左退或统一淡入）。
  - 加载反馈：骨架屏与渐进占位，减少旋转菊花。

## API 约定与后端配合
- 基础：后端提供 `OpenAPI`，前端可用生成器（如 Orval）生成类型安全客户端。
- 响应包：`{ code: number, message: string, data: any }`；区分业务错误与网络错误。
- 鉴权：JWT 放于 `HttpOnly` Cookie，配合 CSRF 防护；提供刷新接口与登录态检测。
- CORS：后端允许前端域名与必要头部；生产通过反向代理统一域。
- 性能：分页与搜索后端实现；前端缓存热点数据并做去重请求。

## 页面规划（MVP）
- 底部 Tab：`首页` / `工具` / `日程` / `我的`。
- 首页：快捷入口卡片、最近使用、搜索。
- 工具页：网格分组（记账、维修、保养、待办、成员）。
- 日程页：家庭日历、提醒与维护计划。
- 我的：账户、设置、主题与关于。

## 工程与交付
- 开发：TypeScript 全量、路由懒加载、组件按需、图标按需（Iconify）。
- 样式：UnoCSS/Tailwind 建立设计令牌（颜色、圆角、阴影、间距）。
- 质量：ESLint + Prettier、单元测试（可选 Vitest）、类型检查。
- PWA：缓存静态资源、预缓存关键路由、离线兜底页。
- 部署：前端静态资源上 CDN；后端 FastAPI 用 Uvicorn/Gunicorn，Nginx 反代与压缩（gzip/brotli）。
- 监控：前端埋点（PV/事件/错误）、后端日志与慢查询监控。

## 路线图（建议）
- Sprint 1：项目脚手架、主题令牌、首页与底部栏、接口打通登录。
- Sprint 2：工具页与卡片组件、微动效统一、列表与表单基件。
- Sprint 3：日程与提醒、离线兜底、首屏性能优化（LCP/CLS）。
- Sprint 4：深色模式（可选）、细节打磨、无障碍与国际化基线。
